# Интернет-магазин «Web-Larёk»
«Web-Larёk» - это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### IProduct

interface IProduct {
    id: string;
    description: string;
    image: string;
    title: string;
    category: string;
    price: number | null;
} 
Назначение: содержит информацию о товаре.

interface IBuyer {
    payment: TPayment;
    email: string;
    phone: string;
    address: string;
}
Назначение: содержит данные пользователя во время оформления заказа.

interface IGetProductsResponse {
    total: number;
    items: IProduct[];
}
Назначение: ответ сервера, когда запрашиваем товары.

interface IOrder extends IBuyer {
    total: number;
    items: string[];
}
Назначение: формирование данных заказа, которые отправляются на сервер.

interface IOrderResponse {
    id: string;
    total: number;
}
Назначение: содержит ответ сервера в случае успешного создания заказа (Идентификатор заказа + его итоговая стоимость).

interface IProductsLoadedEvent {
    items: IProduct[];
}
Назначение: определение формата данных, когда товары успешно загружены с API.

interface IProductSelectedEvent {
    product: IProduct;
}
Назначение: определение формата данных, когда пользователь выбрать товар.

Методы:

`setPhone()` - сохранение номера телефона.
`setEmail()` - сохранение почтового адреса.
`setAddress()` - сохранение адреса.
`getData()` - возвращает все данные.
`clear()` - очистка формы.
`validate()` - проверка на заполнение полей.

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement, events?: IEvents)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.
`events?: IEvents` - опциональная ссылка на глобальный брокер событий. Используется для коммуникации между компонентами без прямой зависимости.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method?: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<EventName, Set<Subscriber>>` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(eventName: EventName, callback: (event: T) => void)` - подписка на событие, принимает название события и функцию-обработчик. 
`off(eventName: EventName, callback: Subscriber)` - отписка от события, принимает название события и финкцию-обработчик.
`emit<T extends object>(eventName: string, data?: T)` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.
`onAll(callback: (event: EmitterEvent) => void)` - подписывается на все события в приложении. Принимает коллбэк от { eventName, data }.
`offAll()` - сбрасывает все события.
`trigger<T extends object>(eventName: string, context?: Partial<T>)` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Слой представления - view

#### Класс BasketView
Отображает интерфейс корзины: список товаров, общую сумму и кнопку оформления заказа. Реагирует на действия пользователя.

Конструктор класса принимает: `container: HTMLElement` - корневой элемент модального окна корзины и `events: EventEmitter` - шина событий для связи с другими компонентами.

Поля класса:
`basketList` - контейнер для списка товаров.
`totalElement` - элемент отображения общей стоимости.
`orderButton` - кнопка "Оформить", переход к заказу.

Методы класса:
`init()` - инициализирует DOM-элементы и подписывается на клик по кнопке "Оформить".
`set items(items: HTMLElement[])` - обновляет список товаров в корзине, принимает массив DOM-элементов карточек товаров.
`set total(value: number)` - устанавливает отображаемую сумму, принимает число - общую стоимость.
`set canOrder(value: boolean)` - включает/отключает кнопку оформления заказа, принимает флаг доступности заказа.

#### Класс CatalogView
Отображает каталог товаров в виде сетки карточек. Создаёт UI на основе данных и обрабатывает клик по товару.

Конструктор класса принимает: `container: HTMLElement` - элемент, в котором будет отображён каталог и `events: EventEmitter` - шина событий для уведомления о выборе товара.

Поля класса:
`container` - DOM-контейнер для рендеринга карточек товаров.

Методы класса:
`render(items: IProduct[])` - Очищает контейнер и рендерит карточки для каждого товара из списка, принимает массив товаров + доп функция - конвертация изображений у карточек.

#### Класс ContactsView
Отображает и управляет формой ввода контактных данных (email и телефон). Выполняет валидацию в реальном времени и при отправке.

Конструктор класса принимает: `container: HTMLElement` - корневой элемент формы и `events: IEvents` - шина событий для отправки данных при успешной валидации.

Поля класса:
`emailInput` - поле ввода email.
`phoneInput` - поле ввода телефона.
`submitButton` - кнопка отправки формы.
`errorElement` - элемент отображения ошибок.
`isEmailTouched, isPhoneTouched` - флаги, указывающие, взаимодействовал ли пользователь с полями.

Методы класса:
`validateOnBlur()` - выполняет валидацию при потере фокуса. Показывает подходящее сообщение об ошибке, если поля невалидны и были затронуты.
`validate(forceShow = false)` - проверяет корректность email и телефона, обновляет состояние кнопки.
`isValid()` - проверяет, корректны ли оба поля без изменения UI.
`showError(message: string)` - отображает текст ошибки, принимает строку - сообщение.
`hideError()` - скрывает ошибку.
`render(): HTMLElement` - Сбрасывает форму: очищает поля, сбрасывает флаги, прячет ошибки, переопределяет валидацию.

#### Класс Modal
Управляет отображением модального окна: открытие, закрытие, рендеринг контента.

Конструктор класса принимает: `container: HTMLElement` - корневой элемент модального окна и `events: EventEmitter` - шина событий для уведомления об открытии/закрытии.

Поля класса:
`modalContainer` - DOM-элемент модального окна.
`closeButton` - кнопка закрытия (крестик).

Методы класса:
`open()` - открывает модальное окно: добавляет класс `modal_active` и эмитит событие `modal:open`.
`close()` - закрывает модальное окно: удаляет класс `modal_active` и эмитит событие `modal:close`.
`isOpen(): boolean` - Проверяет, открыто ли модальное окно.
`render(content: HTMLElement): HTMLElement` - Рендерит переданный контент внутри модального окна и открывает его, принимает `content: HTMLElement` - содержимое модалки.

#### Класс OrderView
Отображает и управляет формой оформления заказа: выбор способа оплаты и ввод адреса. Выполняет валидацию и передаёт данные дальше.

Конструктор класса принимает: `container: HTMLElement` - корневой элемент формы и `events: IEvents` - шина событий для отправки данных при переходе к следующему шагу.

Поля класса:
`addressInput` - поле ввода адреса доставки.
`cardButton` - кнопка выбора оплаты онлайн.
`cashButton` - кнопка выбора оплаты при получении.
`nextButton` - кнопка «Далее».
`errorElement` - элемент отображения ошибок.
`payment` - текущий выбранный способ оплаты.
`isAddressTouched` - флаг, указывающий, взаимодействовал ли пользователь с полем адреса.

Методы класса:
`validateOnBlur()` - валидация при потере фокуса. Показывает соответствующую ошибку, если не выбран способ оплаты или не указан адрес.
`setPayment(value: TPayment)` - устанавливает способ оплаты, обновляет визуальное состояние кнопок (активная/неактивная), запускает валидацию, принимает тип оплаты.
`validate(forceShow = false)` - проверяет, выбран ли способ оплаты и указан ли адрес. Обновляет состояние кнопки «Далее».
`isValid(): boolean` - проверяет, заполнены ли обязательные поля.
`showError(message: string)` - Отображает сообщение об ошибке, принимает строку - текст ошибки.
`hideError()` - скрывает сообщение об ошибке.
`render(): HTMLElement` - Сбрасывает форму: очищает поля, сбрасывает выбор оплаты, убирает стили активных кнопок, прячет ошибки, переопределяет валидацию.

#### Класс ProductModal
Отображает модальное окно с информацией о выбранном товаре и управляет его состоянием (кнопка "Купить"/"Удалить из корзины").

Конструктор класса принимает: `container: HTMLElement` - корневой элемент формы, `events: EventEmitter` - экземпляр для управления событиями приложения и `cartModel: Cart` - модель корзины для проверки наличия товара.

Поля класса:
`_container` - корневой элемент модального окна.
`title, image, category, description, price, button` - DOM-элементы внутри шаблона, найденные по селекторам.
`_data` - объект с данными товара (IProduct), устанавливается через сеттер.

Методы класса:
`set data(value: IProduct)` - устанавливает данные товара, обновляет интерфейс: заголовок, изображение, категорию, описание, цену, состояние кнопки.
`updateButtonState()` - обновляет текст и состояние кнопки в зависимости от наличия товара в корзине.
`get data(): IProduct` - возвращает текущий объект товара.
`get element(): HTMLElement` - возвращает корневой элемент модального окна.

#### Класс SuccessView
Отображает экран успешного оформления заказа с указанием суммы и кнопкой для закрытия модального окна.

Конструктор класса принимает: `container: HTMLElement` - корневой элемент формы и `events: IEvents` - экземпляр EventEmitter для управления событиями (закрытие модалки).

Поля класса:
`totalElement` - элемент, отображающий сумму заказа.
`closeButton` - кнопка для закрытия модального окна.

Методы класса:
`set total(value: number)` - устанавливает текст с общей стоимостью заказа, принимает value - общая сумма заказа в синапсах.
`render(data?: ISuccessData): HTMLElement` - вызывает родительский render, устанавливает общую сумму заказа, принимает data - объект с полем total.

### Презентер

#### Класс AppPresenter
Центральный класс приложения, реализующий паттерн MVP. Управляет логикой взаимодействия между моделями и представлениями через события, координирует поток данных и отображение интерфейса.

Конструктор класса принимает: `products: Products` - модель каталога товаров, `cart: Cart` - модель корзины, `order: Order` - модель данных заказа, `events: EventEmitter` - шина событий для коммуникации между компонентами и `api: AppApi` - cервис для работы с API (загрузка товаров, отправка заказа).

Поля класса:
`modal` - экземпляр Modal, управляет отображением модальных окон.
`basketModalOpen` - флаг, указывает, была ли открыта корзина (для восстановления при обновлении состояния).

Методы класса:
`private openBasket()` - Открывает модальное окно корзины: рендерит список товаров, общую сумму и кнопку оформления.
`private init()` - Инициализирует приложение: создаёт модальное окно, подписывается на события, загружает товары, настраивает обработчики.
